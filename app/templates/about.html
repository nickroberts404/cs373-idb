Data Source Scraping Process
Marvel Developer Portal

Our data was scraped from http://gateway.marvel.com.

To use the API, Marvel uses an authentication scheme where credentials are passed as URL parameters. First, a developer must obtain a public/private key pair generated by Marvel. To make a request, a timestamp must be generated and passed as parameter "ts," the public key is passed as parameter "apikey," and an MD5 hash of the timestamp and public and private keys is passed as "hash." The authentication scheme exists to allow Marvel to regulate the number of API calls made by the entity possessing a particular key pair.

The purpose of the timestamp is to prevent a hypothetical attacker from copying a public key and hash combination and reusing the credentials. The addition of the timestamp means that the hash will change for each request. In practice, however, the timestamp can be any string and the Marvel API does not reject different requests made with the same timestamp, key, and hash combination.

For Phase 1, we just hacked together a bash script to get the JSON response and manually extracted the relevant data. We included this script in our GitHub repository.

For Phase 2, we developed a Python module, marvelpy, that uses the Python Requests library to make HTTP requests. marvelpy presents a simple set of functions to access API resources while taking care of the preparation and passing of credentials. Since Marvel only returns up to 100 resources (comics, characters, etc.) per call, an iterator construct was added to marvelpy to abstract away the housekeeping involved in making repeated requests.

Once scraped, the data from the JSON files was added to our database.
